== Overview

This is a comprehensive study guide for **intermediate Java developers** preparing for **technical interviews and professional-level review**. The notes focus on understanding core concepts deeply while enabling quick recall of key features.

Each Java version is documented in its own file with:

- **Deep-dive coverage** of major features
- **Collapsible sections** for focused study
- **Real-world examples** with interview-style scenarios
- **Best practices** and common pitfalls
- **Quick review questions** for self-assessment

== Study Approach

**Recommended Path:**

1. Start with Java 8 (foundation for modern Java)
2. Progress chronologically through versions
3. Focus on features relevant to your interview preparation
4. Review example code and run scenarios locally
5. Use quick review questions to test understanding

**Time Investment:**

- Deep dive per version: 2-4 hours
- Quick review per version: 30-45 minutes
- Example code practice: 1-2 hours per version

== How to Use This Guide

**For Interview Preparation:**

1. Focus on LTS versions first (8, 11, 17, 21)
2. Read each feature's "Overview" and "Why use it" sections
3. Study code examples and understand the patterns
4. Review "Common pitfalls" - these often appear in interviews
5. Test yourself with "Quick Review" questions

**For Quick Reference:**

1. Use collapsible sections to jump to specific features
2. Review "Best practices" for production code guidance
3. Check "When not to use" to avoid anti-patterns

**For Hands-On Practice:**

1. Navigate to `examples/javaX/` directories
2. Run the code locally (requires appropriate JDK version)
3. Modify examples to experiment with variations
4. Create your own scenarios based on the patterns
