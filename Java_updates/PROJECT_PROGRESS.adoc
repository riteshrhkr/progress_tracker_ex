== Content Strategy

**Core Principles**:
1. *Prioritize by practical importance* - Focus depth on features used daily in real development
2. *Skip or minimize rarely-used features* - Low depth for concepts that take time to learn but don't affect daily work much
3. *Consolidate minor improvements* - Cover related improvements together even if from different versions
4. *Flexibility on version accuracy* - This is for understanding, not strict historical reference
5. *Avoid redundancy* - Don't create separate sections for features already well-covered

**Depth Guidelines**:
- *High depth*: Daily-use features (var, Records, Virtual threads, String methods, Pattern matching)
- *Medium depth*: Useful but not universal (HTTP Client, Switch expressions, Text blocks)
- *Low/Skip*: JVM internals, GC improvements, preview features, rarely-used APIs

**LTS Focus**: Deep coverage for Java 8, 11, 17, 21, and 25 (LTS versions)

== Migration Recommendations

**From Java 8**:
- → Java 11 (LTS) - Stable, widely supported
- → Java 17 (LTS) - Modern features, current standard
- → Java 21 (LTS) - If ready for Virtual Threads

**From Java 11**:
- → Java 17 (LTS) - Recommended for most
- → Java 21 (LTS) - For modern concurrency needs
- → Java 25 (LTS) - Early adopters only (2025-2026)

**From Java 17**:
- → Java 21 (LTS) - When ready for Virtual Threads
- → Java 25 (LTS) - Latest LTS (from 2026+)

**From Java 21**:
- → Java 25 (LTS) - When mature (2026-2027)

**Skip**: Java 22-24 (non-LTS transitional releases)
